Detailed Description

Class Definition CClientRequestResponse for LLRP parameter ClientRequestResponse.

Document Reference LLRP Specification Section 11.2.2

Document Reference LLRP Specification Section 16.2.5.1.3.1 

<p>This parameter describes the list of OpSpecs that the Reader has to execute on the tag for which a Client request was initiated. The AccessSpecID is the identifier of the AccessSpec that had the Client request; the EPC data is the singulated data of the tag for which this Client request was initiated. The AirProtocolOpSpec list contained in the ClientRequestResponse SHALL be processed as the next OpSpecs sent over the air interface. If the AirProtocolOpSpec List is empty, then the execution of the AccessSpec specified by AccessSpecID is cancelled.</p>

void LLRP::CClientRequestResponse::assimilateSubParameters ( CErrorDetails *  pError)
virtual

Assimilates the decoded sub-parameters into m_listAllSubParameters.

After fields are decoded, the CDecoder itself takes care of gathering the subparameters into m_listAllSubParameters. Once the end of the enclosing TLV (or message) is reached this assimilateSubParameters() function is called to create parameter refrences from the primary member variables.
